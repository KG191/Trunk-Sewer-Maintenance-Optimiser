Methodology Overview

Understanding the optimization approach and mathematical framework

Algorithm Overview

This application uses a Risk-First Optimization Algorithm with dynamic weighting to prioritize trunk sewer maintenance. The system balances:

  • Risk Priority: High-risk assets scheduled first regardless of operational factors
  • Geographic Efficiency: Assets within 500m radius grouped for coordinated maintenance
  • Operational Optimization: Access, bypass, and timing considerations within risk tiers

Scoring Formula

Total Score = (Risk Score √ó Risk Weight) + (Operational Score √ó Operational Weight)
Risk Score: Based on asset condition and consequence (20-100 points)
Operational Score: Access + Bypass + Work Scope + Timing (0-67 points)
Weights: User-configurable (default: 60% Risk, 40% Operational)

Algorithm Steps

  1. Calculate Scores: Compute risk and operational scores for all assets
  2. Risk-First Sorting: Group assets by risk tier (High > Moderate > Low)
  3. Within-Tier Ranking: Sort by total weighted score within each risk tier
  4. Geographic Clustering: Identify assets within 500m proximity
  5. Capacity Allocation: Schedule assets respecting annual length constraints
  6. Proximity Coordination: Include lower-risk assets if near high-risk assets

Key Assumptions & Limitations

Model Assumptions

  • Risk ratings remain constant over planning period
  • Weather and seasonal factors not considered
  • Asset deterioration follows linear progression
  • Bypass availability is guaranteed when required
  • Geographic coordinates are accurate and up-to-date

System Limitations

  • Maximum 1000 assets per optimization run
  • 20-year maximum planning horizon
  • Requires Google Maps API for accurate proximity detection
  • RevI data format specific (excludes network connectivity)
  • Static access and bypass scoring assumptions

Data Requirements

The system requires CSV data with these essential columns for RevI format:

  • Asset ID: Unique identifier for each trunk main
  • Length (m): Asset length in meters
  • Size: Pipe diameter (DN150, DN375, etc.)
  • Risk Rating: Low Risk, Moderate Risk, High Risk
  • Access score: Numerical access difficulty (1-10 scale)
  • Bypass required: Y/N flag for bypass requirements
  • Risk: Numerical risk score (1-10 scale)
  • Latitude and longitude: Geographic coordinates
